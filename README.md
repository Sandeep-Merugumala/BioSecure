**BioSecure** is a secure, web-based Healthcare Management System designed with a **"Security First"** approach. By integrating advanced cryptographic primitives‚Äî**AES-256 Encryption**, **RSA Digital Signatures**, and **Hash Chaining**‚Äîdirectly into the application logic, BioSecure ensures that patient medical records remain confidential, their integrity is verifiable, and all system actions are recorded in a tamper-proof audit log.

---

## üöÄ Key Features

### üîí Security
- **End-to-End Encryption (AES-256):** Sensitive medical records (diagnoses, prescriptions) are encrypted before storage. Even database administrators cannot read the raw data.
- **Digital Signatures (RSA):** Every record is cryptographically signed by the doctor's private key, ensuring **Non-Repudiation** and **Data Integrity**.
- **Tamper-Proof Audit Logs:** A blockchain-inspired hash chain ensures that log entries cannot be modified or deleted without invalidating the entire chain.
- **Secure Authentication:** Robust user authentication using `bcrypt` password hashing.

### üè• Functional
- **Role-Based Access Control (RBAC):** Distinct portals for **Admins**, **Doctors**, and **Patients**.
- **Doctor Portal:** securely create and sign medical records.
- **Patient Portal:** View personal medical history with automatic decryption.
- **Admin Dashboard:** Monitor system integrity and view audit logs.

## ‚öôÔ∏è Installation

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/yourusername/biosecure.git
    cd biosecure
    ```

2.  **Install Dependencies**
    ```bash
    npm install
    ```

3.  **Configure Environment Variables**
    Create a `.env` file in the root directory. You can copy the structure below:
    ```env
    DB_HOST=localhost
    DB_USER=root
    DB_PASSWORD=your_mysql_password
    DB_NAME=ehealth_db
    SESSION_SECRET=your_super_secret_key_for_session
    ```

## üóÑÔ∏è Database Setup

1.  **Initialize the Database**
    Log into your MySQL console and run the commands found in `schema.sql`, or import the file directly:
    ```bash
    mysql -u root -p < schema.sql
    ```

2.  **Seed Default Users**
    BioSecure comes with a seeder script to create dummy doctors, patients, and an admin with valid cryptographic keys.
    ```bash
    node seed_users.js
    ```
    > **Note:** This script generates a default password for all users (e.g., `Sandeep0512@` or similar, check `seed_users.js` logs) and populates the database with encrypted records.

## üõ°Ô∏è Security Implementation Details

This project is a demonstration of how to apply cybersecurity principles in software development:

- **Confidentiality:** Achieved via AES-256-CBC encryption of the `diagnosis` and `prescription` columns.
- **Integrity & Non-Repudiation:** Achieved via RSA-SHA256 digital signatures on record creation. Verification occurs whenever a record is accessed.
- **Accountability:** Achieved via Hash-Chained Audit Logs (`prev_hash + current_data_hash`).
